Mục tiêu:

- Hiểu các thông số cấu hình mạng của máy tính.

- Hiểu chức năng của Gateway và DNS Server.

- Biết sử dụng lệnh tracert trên Windows để kiểm tra đường đi của dữ liệu.

- Mô phỏng hệ thống mạng bằng Cisco Packet Tracer.

- Thực hành các công cụ giao tiếp trực tuyến phổ biến.

Nội dung thực hành

Bài 1: Kiểm tra thông số cấu hình mạng:
Tìm thông tin cấu hình mạng của máy tính đang sử dụng.

Các thông số cần thu thập:
IP address:
Subnet mask:
Gateway:
DNS server:


Bài 2: Chức năng của Gateway và DNS

- Gateway là thiết bị trung gian cho phép các máy tính trong mạng nội bộ giao tiếp với các mạng khác (ví dụ Internet).

- DNS Server có chức năng chuyển đổi tên miền (ví dụ: google.com) thành địa chỉ IP để máy tính có thể kết nối đến máy chủ.

Bài 3: Kiểm tra đường truyền với tracert

Sử dụng lệnh sau trên Command Prompt:
tracert www.ctu.edu.vn
tracert gmail.com

Quan sát các nút trung gian (hops) mà dữ liệu đi qua.

Các trường hợp kiểm tra:

- Kết nối đến www.ctu.edu.vn

- Kết nối đến gmail.com

- Kết nối đến máy tính trong cùng phòng

- Kết nối đến máy tính khác phòng trong mạng trường

Bài 4: Mô phỏng mạng với Cisco Packet Tracer

Yêu cầu:

- Tạo mô hình mạng gồm 2 phòng máy tính, mỗi phòng có: 8 PCs, Switch hoặc Hub, Router, Modem

Sơ đồ mạng:

Sinh viên thiết kế sơ đồ kết nối mạng giữa hai phòng máy bằng Cisco Packet Tracer.
